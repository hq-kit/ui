## Installation via CLI
Before executing the command, ensure that **tailwind-css** is installed. If not, follow the instructions [here](https://tailwindcss.com/docs/installation) for installation.
<CLI command='init' />
or you can choose any theme listed [**here**](/themes)

<Note>
This CLI will overwrite your **.css** file. It is highly recommended to use this in a **new project**.
</Note>

### Add Components
Next, use **` add [component-name] `** to add components in your project. This allows for selective installation without the need for a full setup. Install only what you need, or refer to the documentation to manually include components. Here's how to add a `button`:
<CLI command='add' items={['button']} />

Or add multiple components at once:
<CLI command='add' items={['button', 'select', 'input']} />

## Manual Installation
To begin setting up the design system, make sure `tailwindcss` is already installed. Then, start by installing these packages:

<CLI command='install' items={['react-aria-components']} />

Next, add tailwind-variants and clsx for utility functions:

<CLI command='install' items={['tailwind-variants', 'clsx', 'tailwind-merge']} />

Following that, include the animation package for managing animations and icons:

<CLI command='install' items={['@tabler/icons-react', 'tw-animate-css']} />

### Utils
Create a helper function. Usually located in `lib/utils.ts`:

```ts title='lib/utils.ts'
import { type ClassValue, clsx } from 'clsx'
import { twMerge } from 'tailwind-merge'

export const cn = (...inputs: ClassValue[]) => twMerge(clsx(inputs))
```

### Hooks
Create a use-mobile hook. Usually located in `hooks/use-mobile.ts`:

```ts title='hooks/use-mobile.ts'
import { useEffect, useState } from 'react'

export function useIsMobile(mobileBreakpoint = 768) {
  const [isMobile, setIsMobile] = useState<boolean | undefined>(undefined)

  useEffect(() => {
    const mql = window.matchMedia(`(max-width: ${mobileBreakpoint - 1}px)`)
    const onChange = () => {
      setIsMobile(window.innerWidth < mobileBreakpoint)
    }
    mql.addEventListener('change', onChange)
    setIsMobile(window.innerWidth < mobileBreakpoint)
    return () => mql.removeEventListener('change', onChange)
  }, [])

  return !!isMobile
}
```

### TS Config
Next, incorporate the following settings into your `tsconfig.json` file:
```json {3-6} title='tsconfig.json'
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./*"]
    }
  }
}
```

### TS Config with src Folder
Ensure you configure the `paths` property correctly if your project includes a `src` folder. If you want to organize
the UI folder, adjust your paths accordingly like this:

```json {3-6} title='tsconfig.json'
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```
